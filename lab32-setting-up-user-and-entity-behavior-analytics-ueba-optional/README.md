# SIEM Lab 32 – Setting Up User and Entity Behavior Analytics (UEBA)

## Overview

User and Entity Behavior Analytics (UEBA) helps security teams identify unusual user activities and potential insider threats by analyzing behavioral patterns rather than relying solely on traditional security rules.

In this lab, Elasticsearch, Kibana, Filebeat, and Auditbeat were used to review user activities and identify anomalous behavior.

---

## Lab Objectives

* Understand the fundamentals of UEBA.
* Review user activity logs collected by Elastic Stack.
* Identify unusual user account behavior.
* Analyze authentication and account-related events.
* Interpret behavioral analytics results.

---

## Tools Used

| Tool          | Purpose                  |
| ------------- | ------------------------ |
| Elasticsearch | Data Storage             |
| Kibana        | Visualization & Analysis |
| Filebeat      | Log Collection           |
| Auditbeat     | User Activity Monitoring |
| Ubuntu Linux  | Operating System         |

---

# Task 1 – Verify UEBA Environment

The Elastic Stack services were verified to ensure the environment was operational.

### Commands

```bash
sudo systemctl status elasticsearch

sudo systemctl status kibana

sudo systemctl status filebeat

sudo systemctl status auditbeat
```

### Result

All required services were running successfully.

---

# Task 2 – Generate User Activity

Two test users were created.

### Commands

```bash
sudo useradd analyst1

sudo useradd analyst2
```

User sessions were generated.

```bash
sudo su - analyst1

exit

sudo su - analyst2

exit
```

### Result

User login activity was generated for behavioral analysis.

---

# Task 3 – Simulate Suspicious Activity

A failed login attempt was generated.

### Command

```bash
su fakeuser
```

### Result

Authentication failure events were created and collected by Auditbeat.

---

# Task 4 – Review User Behavior Timeline

Kibana Discover was used to review collected events.

### Important Fields

```text
user.name
host.name
event.action
source.ip
@timestamp
```

### Result

User activities were visible through Kibana and could be analyzed chronologically.

---

# Task 5 – Identify Unusual Activity

## Normal Behavior

* Successful user logins
* Normal command execution
* Standard account usage

## Suspicious Behavior

* Multiple failed logins
* Unusual account activity
* Unauthorized access attempts
* Repeated authentication failures

Reviewing login and account activity.

Recognizing suspicious activities that may indicate compromise.
