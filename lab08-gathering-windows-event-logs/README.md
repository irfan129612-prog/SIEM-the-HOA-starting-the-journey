# 🧪 Lab 08: Gathering Windows Event Logs

## 📌 Lab Summary

In this lab, **Winlogbeat** was installed and configured on a Windows system to collect Windows Event Logs and forward them to the **Elastic (ELK) SIEM** platform. The configuration was validated, the Winlogbeat service was started, and log ingestion was verified through Kibana.

---

## 🎯 Objectives

- Understand Windows Event Log collection
- Install and configure Winlogbeat
- Configure Windows Event Log sources
- Forward logs to Elasticsearch
- Verify log ingestion in Kibana

## ✅ Task 1: Install Winlogbeat

Winlogbeat was downloaded from the official Elastic website and extracted to the following directory:

```text
C:\Program Files\Winlogbeat
```

### Install the Winlogbeat Service

Open **PowerShell** as Administrator and run:

```powershell
cd "C:\Program Files\Winlogbeat"

.\install-service-winlogbeat.ps1
```

---

## ✅ Task 2: Configure Windows Event Logs

The `winlogbeat.yml` configuration file was modified to collect the following Windows Event Logs:

```yaml
winlogbeat.event_logs:
  - name: Application
  - name: Security
  - name: System
```

These logs provide visibility into:

- Application events
- Security events
- System activities

---

## ✅ Task 3: Configure Elasticsearch Output

Winlogbeat was configured to forward logs to the Elasticsearch server hosted on AWS EC2.

```yaml
output.elasticsearch:
  hosts: ["https://AWS_PUBLIC_IP:9200"]
  username: "elastic"
  password: "YOUR_PASSWORD"
  ssl.verification_mode: none
```

> Replace `AWS_PUBLIC_IP` and `YOUR_PASSWORD` with your own environment values.

---

## ✅ Task 4: Validate the Configuration

The Winlogbeat configuration was tested before starting the service.

```powershell
.\winlogbeat.exe test config
```

### Expected Output

```text
Config OK
```

---

## ✅ Task 5: Start the Winlogbeat Service

Start the Winlogbeat service:

```powershell
Start-Service winlogbeat
```

Verify the service status:

```powershell
Get-Service winlogbeat
```

### Expected Status

```text
Running
```

---

## ✅ Task 6: Verify Windows Event Logs

View the latest Application Event Logs using PowerShell:

```powershell
Get-EventLog -LogName Application -Newest 5
```

This confirmed that Windows Event Logs were being generated and available for collection.

---

## ✅ Task 7: Verify Logs in Kibana

Access the Kibana web interface:

```text
http://AWS_PUBLIC_IP:5601
```

Navigate to:

- **Discover**
- Select the appropriate Winlogbeat data view
- Search for incoming Windows Event Logs

Successful log entries confirmed that Winlogbeat was forwarding data to Elasticsearch.
